1. Purpose
This document confirms the scope, delivery plan, pricing, and responsibilities for developing Version 1 of the Aluprofile Search & Catalog System.

2. Scope of Work (Version 1)
The Service Provider will deliver a web application with:

A. Public Area
- Home page with application and cross-section overviews (including profile counts).
- Profile search/list with filters (application, cross-section, supplier, material, dimensions) and text search.
- Profile detail page with drawing, technical data, usage/status, supplier contact, and optional images/logo.

B. Admin Area (Login Protected)
- Admin login/logout.
- Profile management: create, edit, delete, status update, drawing/image upload.
- Supplier management: create, edit, delete with contact data.
- Application and cross-section management: create/edit categories.

C. Quality Requirements
- Responsive design for desktop/tablet/mobile.
- Structure prepared for German/English content.
- Extensible database design.

3. Technology Stack
- Backend: Node.js (NestJS)
- Frontend: React
- Database: PostgreSQL (SQL)
- Data access: Prisma ORM, with selective raw SQL for advanced search/reporting where appropriate.

4. Estimated Effort and Timeline
- Estimated effort: 80-90 developer hours.
- Weekly allocation: approx. 10-12 hours.
- Estimated duration: 7-9 weeks from project start.

Milestones
- Weeks 1-2: final requirements, data model and architecture, initial setup.
- Weeks 3-4: public pages and search/filter implementation.
- Weeks 5-6: admin area, CRUD functions, file upload.
- Weeks 7-8/9: testing, fixes, documentation, deployment support.

5. Fees and Payment Terms
- Hourly reference rate: EUR 25/hour.
- Fixed price for Version 1 scope: EUR 2,100 (excluding VAT, if applicable).

Payment schedule:
- 30% (EUR 630) at project start.
- 40% (EUR 840) after working test version delivery.
- 30% (EUR 630) after final handover and approval.

Work outside the defined scope will be quoted separately and billed at EUR 25/hour unless otherwise agreed in writing.

6. Responsibilities
Service Provider:
- Design and implement the application and database.
- Provide run/deployment instructions.
- Provide up to 2 weeks of post go-live support for minor bug fixes.

Client:
- Provide initial profile/supplier data (Excel/CSV/scans).
- Provide logos, drawings, and photos.
- Provide timely review and feedback during implementation.

7. Hosting and Delivery
Hosting options:
- Client-provided Windows/Linux server or hosting account, or
- Provider-assisted hosting selection (additional billable time).

Deliverables:
- Frontend and backend build/deployable artifacts.
- Database schema and migration scripts.
- Setup and run documentation.

8. Acceptance
The project is considered accepted when:
- Agreed Version 1 scope is delivered,
- Major functional issues are resolved,
- Client provides written approval.
