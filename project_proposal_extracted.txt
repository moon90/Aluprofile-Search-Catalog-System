1. Project Overview 
Oliver would like a simple, modern web application to manage and search aluminium profiles. 
The system should: 
• Show applications (z.B. Maschinenbau, Solarindustrie, …) and cross-sections. 
• List and search profiles with drawing, description, usage and supplier contact. 
• Provide a detailed profile page with all technical data. 
• Offer a secure admin area where authorised users can add and update profiles. 
The goal is a practical business tool, not a complex enterprise system. It should be easy to use for 
non-technical users and easy to extend later. 
 
2. Main Features (Scope of Version 1) 
2.1 Public Area (for all visitors) 
1. Home page 
o List of application areas (Anwendungen) with the number of profiles per category. 
o List of cross-section types (Querschnitte) with the number of profiles. 
o List of newest / highlighted profiles with small drawing and key data. 
2. Profile search & list 
o Filter by application, cross-section type, supplier, material and dimensions. 
o Text search by profile name or description. 
o Table view similar to the printout you provided (drawing, description, usage, photo, 
supplier). 
3. Profile detail page 
o Technical drawing. 
o Dimensions, weight per meter, material, length. 

o Usage description and status (e.g. “vorrätig”). 
o Supplier contact details (address, phone, e-mail). 
o Logo and optional pictures. 
2.2 Admin Area (protected by login) 
4. User login / logout 
o Password-protected access for you and future staff. 
o Role “Admin” for full access; further roles can be added later. 
5. Profile management 
o Create, edit, delete profiles. 
o Upload drawing (image/PDF) and application photo. 
o Set application categories and cross-section type. 
o Set status (available / in development / not available). 
6. Supplier management 
o Create, edit, delete suppliers. 
o Store address, contact person, phone, e-mail, website. 
7. Application & cross-section management 
o Create and edit application categories. 
o Create and edit cross-section types. 
2.3 Non-functional requirements 
• Responsive web design (usable on PC, tablet and phone). 
• German/English ready (texts can be stored so translation is possible later). 
• Clean database design so the system can be extended in future. 
 
3. Technology (short and non-technical) 
• The system will be built with Microsoft .NET (current version) and a SQL database. 
• This is widely used enterprise technology, stable and easy to host on Windows or Linux 
servers. 
• For you as a user there is no technical knowledge required: you will access the system via a 
normal web browser (Chrome, Edge, Firefox).
